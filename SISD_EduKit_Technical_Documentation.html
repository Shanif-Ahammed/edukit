Executive Summary

SISD EduKit - Teacher Portal is a high-performance offline-first workspace designed specifically for educators at the Swiss International Scientific School in Dubai (SISD). The portal provides tools for student roster ingestion, classroom seating charts, cohort analytics, and localized reports drafting.

In educational environments, security requirements are exceptionally rigorous. Traditional cloud applications require roster spreadsheets and student records to be uploaded onto remote servers, exposing the institution to massive data breach vectors. SISD EduKit resolves this conflict entirely by using a **pure client-side, 100% backend-less model**. The entire system operates without a database backend, file server, or application backend. All parsing, calculations, seating arrangements, and report comments assembly take place inside the memory of the teacher's browser sandbox on their physical device.

To ensure complete data privacy and grading integrity, the report card comment builder is **purely coded client-side** using a **pre-authored JSON comment bank** written by subject coordinators and Senior Leadership Team (SLT) members. The system dynamically stiches segments together based on student grades and progress, requiring no cloud-based AI networks, database storage, or external server processing. No roster data is ever transmitted across the internet, stored on a remote server, or backed up externally.

Core Privacy Pillars (Zero-Storage Guarantee)

The portal's data protection strategy is built around four rigorous safeguards that ensure complete data sovereignty remains with the individual teacher and SISD:

1. Zero Data Transmission

Rosters and grade files imported from iSAMS are parsed fully in memory locally. Zero bytes of student data are sent over networks or public cloud layers.

2. Browser Sandbox Storage

Operational states are retained in standard isolated browser LocalStorage. Only pages running on the exact same domain can view the records.

3. Instant Local Purging

Teachers can delete all rosters, grades, and comments instantly by clicking the "Disconnect MIS Data" button, which erases all keys from LocalStorage.

4. Stateless & Database-Less

The entire web application has no server database, backend runtime, or cloud files storage, removing any possibility of system breaches.

⚠️ Crucial Local Boundaries & Data Integrity:
  • Purely Coded Local Assembly: The report comment generator is a purely coded client-side builder that maps criteria scores directly against pre-authored JSON comment banks written by subject coordinators and SLT. Because no network calls, AI APIs, or databases are connected to compile comments, roster records are structurally protected from online exposure.
  • Direct Excel Exclusion: The interactive help chatbot assistant (`AiAssistant`) has no direct access to your attached Excel spreadsheets. The connected spreadsheet lives entirely inside your local browser memory variables and is never transmitted to Vercel or Google.
  • Zero Model Training: Chatbot transactions strictly consist of generic, anonymized user guides and system instructions. Requests route through Google's developer API tier, which has no access to student data or portal source code for AI model training purposes (prompt inputs are never saved, logged, or utilized to train Google models).

1. Technology Stack

The application is engineered strictly with lightweight, industry-standard web components. This ensures rapid initial loading, platform independence, and reliable long-term maintainability.

  • React 19 & Vite 8: Powers dynamic client-side rendering with instant reload properties. No heavyweight server framework runtime is loaded.
  • SheetJS (xlsx): Handles parsing of binary Excel exports directly in the browser's thread, mapping columns instantly.
  • Service Worker (sw.js): Implements progressive PWA capabilities, caching app templates to allow offline initialization.
  • Vanilla CSS Custom Variables: A customized, highly reactive design layout supporting glassmorphism and clean printing overlays.
  • Lucide Icons & Canvas Confetti: Renders vector icons and lightweight particle effects utilizing hardware-accelerated HTML5 canvases.

A. Stateless Help Chat Proxy

To support an interactive technical support assistant (AiAssistant) to help teachers troubleshoot Excel templates, the portal features a stateless, zero-key cloud proxy:

  1. The static React app is hosted on GitHub Pages. When loaded, it accesses a serverless function hosted at /api/gemini.js on Vercel.
  2. The Vercel environment variable securely contains a private API key, acting as a stateless gateway for general educational help prompts.
  3. The Help Assistant is completely separate from active gradebooks and seating charts. It cannot access connected Excel rosters or local browser databases. Teachers query the chatbot strictly for user guide assistance, with zero network transaction of student records.

B. Service Worker Caching Strategy

To support offline access on campus, the custom service worker (sw.js) utilizes two primary request management schemes:

  • Pre-caching: Automatically caches the index file, styling frameworks, and key IB grading rubric json documents during app install.
  • Stale-While-Revalidate: Static assets are pulled immediately from the local device cache for millisecond load times, while a background thread queries the server to update the cached assets for subsequent visits.

The system reduces typical web application vulnerability vectors by maintaining a stateless backplane.

6. Quality Assurance (QA) & Reporting Cycle Audit

The SISD EduKit acts as a core technical enabler for the official school 2025-2026 Academic Data Framework Quality Assurance (QA) Guidelines. The portal structurally aligns system controls with institutional review roles to guarantee grading and reporting integrity:

  • Subject Coordinators (SubCos) Review Pathway:
    • Missing Data Auditing: If a student connected from iSAMS is missing any required MYP criterion grades (Criterion A, B, C, or D), the system locks down comment generation, labels the student ⚠️ Missing Grades, and halts copying. This assists SubCos in chasing teachers to input missing assessment data in the Online Assessment System (OAS).
    • Comment Bank Alignment: SubCos can review, test, and amend pre-authored JSON comment banks directly within the browser, verifying alignment with academic standards before AP5 releases.
  • Teacher Peer-Checking Interface:
    • Grade Boundary Guard (1/2 Marks): System logic flags border grade boundary totals (1 or 2 points below higher grading tiers), warning teachers to re-evaluate attainment before finalizing entries in iSAMS.
    • Manual Draft Interception: If a student achieves a final MYP Grade of 1 or 2, or individual criteria grades of 1 or 2, the system automatically intercepts auto-generation, labels the row ⚠️ Manual Draft, and restricts default stitching. This prompts teachers to draft custom support plans and intervention comments.
  • Inclusion, EAL, and MAGT Spot-Checking:
    • Real-Time Balancing Badges: The Seating Chart Studio and Cohort Analysis tools display colored badges showing EAL, Emirati, Inclusion (SEN), and MAGT allocations, allowing specialist coordinators to audit and verify individual student groups directly on A4 grids.
  • Data Manager Framework Audits:
    • Attainment vs. KHDA Judgments: The Cohort Analysis Dashboard processes average criteria achievement scores (Crit A-D) and gender distributions, allowing the Data Manager to run local validations against KHDA attainment boundaries (Outstanding, Very Good, Good, Acceptable) for AP5 reporting, keeping all computations local.

7. Core Benefits & Performance Analysis

Because calculations occur locally in native client processes, operations bypass server latency. Processing a roster of **1,000+ students takes under 50 milliseconds**.

C. Institutional Stability

If external networks suffer downtime, the application remains fully functional offline, allowing teachers to work uninterrupted.

8. Maintenance & Support Framework

The static serverless design of SISD EduKit shifts operational liabilities from academic IT departments to automatic client-side execution boundaries:

  • Zero Server Liabilities: Bypasses typical OS security patching, database backup maintenance, and resource scaling rules. The site operates on a 100% static file structure.
  • Seamless Background Updates: Service worker caching (sw.js) manages asset invalidation, silently updating layout shell components on subsequent loads when a new build is detected.
  • IT Troubleshooting Protocol: Since the application carries no database states, 99% of user issues are resolved instantly by clicking "Disconnect MIS Data" or doing a hard refresh (Ctrl + F5), resetting browser storage buffers with zero risk of database corruption.
  • Standard Browser Portability: Native compatibility with Google Chrome, Microsoft Edge, Mozilla Firefox, and Apple Safari.
